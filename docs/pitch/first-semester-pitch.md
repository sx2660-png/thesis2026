# First-semester Pitch

This page will document the first-semester presentation as the prototype and evaluation develop.

## Planned content

- The problem and the target trading workflow.
- A walkthrough of the current prototype in use.
- What triggers a warning and how the evidence is shown.
- Where the trader can confirm, override, recalculate, or pause the workflow.
- What was learned from testing and feedback.
- Current limitations and the next stage of the thesis.

The final presentation should show the design in use rather than only describe the agent architecture. It should also distinguish what the prototype demonstrates from what would still require enterprise data, user testing, or production integration.
